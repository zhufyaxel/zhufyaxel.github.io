## Front Matter
```
---
layout: post
title: "New title"
date: 2017-08-20 15:58:50 -0700
categories: jekyll Cat2
---
```
* 主要是用于生成Post的相关信息，支持YAML&JSON
* url rules:
  keywords in categories/date in front matter/title in file name
* 注意变量不仅仅是如上一些，定义是在Layout类里的。例如Post还能增加Author

# Posts
可以写HTML的，不过我应该不会去弄
.md里，以`layout: "post"`定义其属性。一旦定义完成，文件名里的日期和Title会被拿来直接用
所以实际上在`Front Matter`里的定义是个`Overwrite`，但是URL里面比较迷，主要看上方
很有意思的是，`_post`里面的dir不会被用于URL，所以归类整理Post的Folder可以随意搞

# Pages
layout: page
link会直接弄到/pagename

`permalink`可以用于Page, post以及其他，都可以搞
也可以动态。例：
```
permalink: /:categories/:year/:month/:day/:title.html
```

# Default Front matter
总归每次自己打还是麻烦
定义可以在config.yml里面搞
这边已经定义成了Post, 代码如下:
```YML
defaults:
  -
    scope:
      path:  "_posts/TEST" #路径可以留空，会变成全局
    values:
      layout: "post"
      categories : test jekyll
```

# layouts
1. Define a folder in `_layouts`
2. Using a name of "name.html"，然后搞自己想搞的东西
3. 在需要插入内容的地方，用`content`
4. 嵌套逻辑：
  1. 先定义A模板
  2. 在B模板调用A模板即可
* Navigation就用这个套路来做

# loop navigation
